# Devlog — week 1: tracer bullet

2026-07-05

## What shipped

The goal for week 1 was a tracer bullet: the thinnest end-to-end slice of
the product, real at every layer. It exists. `POST /v1/verify` takes an
answer plus context passages, scores groundedness with a real model, and
returns a verdict; it runs in a container, deploys to a kind cluster via
the Helm chart, and every PR runs lint, secret scanning, the test suite,
and an e2e smoke request against the deployed chart.

Around the bullet, the scaffolding that makes the next weeks cheap:
repo bootstrap, the agent constitution, OpenSpec for feature contracts,
issue and PR templates, the CI pipeline, and six ADRs recording the
decisions that shaped the week (`docs/adr/`).

The week also contained its first swerve. The tracer bullet shipped on
HHEM-2.1-open, but the spike that integrated it surfaced enough
operational red flags — frozen upstream since mid-2024, `trust_remote_code`,
a second Hub repo fetched at runtime, a permanent `transformers<5` pin —
that the signal-model question got a proper benchmark instead of a shrug.
Five candidates ran on a common RAGTruth slice under the laptop-CPU
protocol (`evals/RESULTS.md`); LettuceDetect won on both accuracy and the
operational bar, ADR-0006 recorded the decision, and the migration to
`lettucedetect-v2-mmbert-base` landed before the week closed. The
`transformers<5` pin left with HHEM.

## Spike findings

Two timeboxed spikes settled the architecture question
(`docs/spikes/`). The FastAPI + HHEM spike established the shape of the
cost: inference is 20–140 ms per claim/evidence pair on laptop CPU and
dominates everything else, and the model process carries a ~500 MB
resident set wherever it lives. The Go-front + Python-sidecar spike
priced the split architecture: the localhost hop costs ~0–1 ms — noise
against inference — while the seam costs a request contract defined
twice with no compiler spanning it, plus a second process lifecycle to
babysit in dev and CI. Go's genuine virtues (sub-second builds, 12 MB
RSS, instant start) sat next to a model process that dwarfs them either
way.

## The language decision

Plumb's API and engine are Python — one process, one language
(ADR-0001). The spikes converged on this from both directions: the split
buys nothing at the bottleneck, since inference dominates and the model
process exists in both architectures, and it costs a permanent seam,
since every future signal would add another duplicated schema across the
Go/Python boundary. Everything on the engine roadmap — decomposition,
retrieval, signals, aggregation, calibration — is model-adjacent and
lands on the Python side regardless, and the ML ecosystem pulls the same
direction. If a hardened edge is ever warranted, a Go or Envoy front can
be added in front of the stable HTTP contract later; that would be an
additive deployment decision, not a reopening of this one.

## Next week

Week 2 is about the word the one-liner stakes everything on:
*calibrated*. Today the response carries a raw model score; the gap
between that and a verdict someone can gate a deploy on is the
trust-critical core of the product. The sequencing is dictated by #29:
first the tier-2 response-shape ADR — span-based attribution, joint
all-passages inference matching the benchmarked configuration, and a
migration path that retires `evidence_index` — then its implementation,
then the calibration methodology, in that order so calibration doesn't
lock onto a scoring mode the ADR is about to retire.
